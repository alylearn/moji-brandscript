🎯 受众锁定 4个核心子群 ▼
MOJi核心用户：18–35岁、正在学或想学日语的中文用户。每期选题先判断面向哪个子群——
  • 入门/零基础：焦虑感强，被「快速入门」「不用死记硬背」驱动
  • 备考党（N5–N1）：目标明确有压力，被「效率」「不忘词」「精准刷题」驱动
  • 兴趣学习者：喜欢日本文化/动漫/影视，被「沉浸式」「真实场景」「开口说」驱动
  • 留学/工作向：有实际使用需求，被「到日本用得上」「实战口语」驱动
✂️ 六种选题切角 点击各切角查看说明 ▼
1
反转式
高点击
▼
否定用户已有认知或习惯，制造「我一直做错了？」的瞬间。答案在视频里，钩子必须反常识。
逻辑模板
「用户以为X是对的」→「其实X是错的/有更好方式」→「MOJi是那个更好的方式」
选题例子
别用眼睛背日语单词——耳朵才是主力
说错了又怎样？这里没有人评判你
适用产品
MOJi辞書（记单词方法）、MOJiKana（写法）、MOJi会话（开口练习）
2
干货式
收藏高
▼
直接给有用信息，用户看完有收获感愿意收藏。标题必须带具体数字或可操作性，不能模糊。
逻辑模板
「[X个方法/步骤]」→ 逐条展示功能 → 结尾可收藏复用
选题例子
N3备考3步走：MOJi这样用才不浪费
MOJi辞書5个隐藏功能，90%用户不知道
适用产品
MOJi辞書（词单/背词演示）、MOJi阅读（阅读流程）、MOJiTest（刷题技巧）
3
痛点式
转化强
▼
精准戳中用户学日语过程中某个具体痛苦，再给出MOJi的解决方案。痛点越具体，共鸣越深。
逻辑模板
「[具体痛苦场景]」→「你不是一个人」→「MOJi帮你解决了这个」
选题例子
背了三遍还是忘——你可能缺这一步
不敢开口说日语？AI陪你练到敢为止
注意
痛点必须真实，不能夸大或虚构，需基于产品实际功能
4
场景式
代入感强
▼
把用户放进具体生活/学习场景里，感受「这说的就是我」。场景要真实可辨认，画面感要强。
逻辑模板
「[你在哪里 + 遇到了什么]」→ 功能出现 → 「问题解决了」
选题例子
在日本餐厅，菜单看不懂怎么办
通勤20分钟，怎么把日语塞进去
适用产品
MOJi会话（旅游/留学场景）、MOJi阅读（通勤/睡前）、MOJi辞書（查词场景）
5
测评诊断式
让用户自己撞见短板
▼
不直接说用户哪里弱，而是用一个测试/诊断让用户自己测出来，激起「我居然这么差？」的不甘。结论由用户自己得出，比你说更可信。
逻辑模板
「敢不敢测一下你的X」→ 用户测出意外结果 → 「想补就用MOJi」
选题例子
你的日语词汇量，到底有多少"水分"？
不靠汉字，你的词汇量还剩多少
适用产品
MOJi辞書（词汇量测试）、MOJiTest（水平诊断）。注意：测试要真能测出落差，不能是走过场
6
汇总干货型
数字 + 合集，靠收藏/搜索拉量
▼
「8个___必会方法」「5个隐藏技巧」这类数字+合集型干货。跟前 5 种切角不同——它不靠单一痛点钩住快速浏览的人，靠的是被搜到 + 被当教程收藏。

参考来源：哈尔的数码屋等跨品类合集账号。MOJi 还没出过同类内容，先跑两条看自己的数据，再回来校准这套写法。
两种子型，选一种用
A. APP 拉新型
通用方法占大头，MOJi 作为「这个方法用 MOJi 怎么落地」出现 1-2 次，位置放前面或中间都行，不必卡末段。
目标：被搜「日语背单词方法」「自学日语」的人搜到，拉到还没用 MOJi 的人。
例：8个日语背单词必会方法 / 5个 N1 备考方法
B. APP 复购/留存型
整条视频几乎都是 MOJi 自己的功能或用法，节奏一致、不单独推销。
目标：让已用 MOJi 的人更会用、提粘性，常放置顶。
例：5个 MOJi 必会隐藏技巧 / 8个 MOJi 辞書老用户都不知道的功能
标题风格
特别适合用「教程/搜索型」标题（数字+教程承诺 / 模拟用户搜索疑问 / SEO 关键词堆叠）—— 具体写法和例子参见上面「选题主线原则 → 标题风格」小节。
脚本结构参考（拆解来源：哈尔的数码屋，单条样本仅供参考）
每条节奏均匀，不要某条特别长或特别快；数字不一定前 3 秒报总数，标题占位即可，结尾回扣；可以不强 CTA，合集属性自动驱动收藏；植入随子型走，都用「直接演示功能」嵌进去，不要单独推销段。
📌 选题主线原则 核心 / 三链一致 / 标题风格 / 选题前检查 ▼
一期视频，要有一个清晰的核心
核心必须在选题阶段锁定，且只有一个：
· 痛点型：一个功能、一个痛点
· 汇总型：一个合集主题（里面 3-8 条都在支撑这个主题）
多功能并列 = 用户记不住任何一个；合集主题模糊 = 用户找不到收藏理由。

判断方法：能否用一句话说出这期的核心？
✓「这期讲 MOJi辞書 的影子跟读，解决发音不准」（痛点型）
✓「这期讲 5 个 N1 必会的 MOJi 隐藏技巧」（汇总型）
✗「这期讲查词、收藏、背词、AI 问答和影子跟读」（没有中心，既不是单一功能也不是合集主题）
选题定封面，封面定调，标题在两种风格里挑
三者要在同一条叙事线上：
反转式选题 → 行为质疑/悬念质疑型封面 → 标题接封面钩子推进
痛点式选题 → 痛点+回报型封面 → 标题先共鸣再给解法
测评诊断式选题 → 测试结果型封面 → 标题抛测试问题，引人想知道答案
汇总干货型选题 → 数字+合集型封面（优先实拍真机） → 标题用教程/搜索型

标题不被封面严格决定——同一个封面下，钩子型和教程/搜索型两种标题都能配，详见下面「标题风格」。
标题风格（两种，可跨切角混搭）
标题不只一种写法——任何选题切角都能在这两种风格里挑，按目标（钩住快速浏览的人 vs 被搜到/被收藏）选：

① 钩子型（适合快速浏览场景） —— 质问、承认、结果先行，靠情绪和冲突钩住用户：
· 「承认了！MOJi 默认背词在浪费你时间」
· 「是你一直在用最没用的方式背」

② 教程/搜索型（适合被搜到/被收藏场景） —— 三种写法都行：
· 数字+教程承诺：「1招教会你怎么自定义日语词单」
· 模拟用户搜索疑问：「MOJi 辞書到底怎么创建日语词单啊？」
· SEO 关键词堆叠：「日语学习 自制词单 使用方法」

痛点型选题默认走钩子型，但想要搜索/收藏拉量时可换教程/搜索型；汇总型选题更适合教程/搜索型。两种风格不绑死切角。
选题前检查
  • 内容基于真实产品功能，不夸大不虚构
  • 目标受众是哪个子群，切角对他们有没有击中
  • 核心能用一句话说清楚（痛点型 = 一个功能；汇总型 = 一个合集主题）

方法论参考
⚡ 叙事结构：痛点直给型（默认） 只要能找出一个单一尖锐痛点就用它 · 数据最好的三条都是这个结构 ▼
复盘结论（三条高数据视频 vs 两条低数据视频）：高数据的全是「钩子 → 痛点直给 → 连续操作演示 → 具体CTA」的轻结构；低数据的全是 AIDA 逐步铺垫或多功能总览。只要本期能拎出一个单一尖锐痛点当入口，就默认走这一套（哪怕涉及多个功能，也用这一个痛点把它们串起来），不要套 AIDA 的「兴趣放大 / 欲望铺垫」中段。
钩
前3秒 · 直接进
钩子——6种可选，挑一种直接进
第一句就进内容，不铺垫、不报数据、不下产品定义、不寒暄。
① 互动指令：「这个发音，你记住了吗？」「来，跟我背一个词」
② 质问否定：「别用最没用的方式背单词了」「你还在死记硬背？」
③ 结果先行：「谁懂啊，背词终于不忘了」「太爽了吧」
④ 对比反差：「之前背一个忘一个，现在看一遍就记住」
⑤ 身份标签：「给N2怎么都过不了的人」「备考党看这条」
⑥ 数字利益：「3个设置改完，背词效率翻倍」「1分钟学会」

✗「有20%的用户……」「其实就是一款日语学习RPG」「大家好」（数据/定义/寒暄慢热）
痛
紧接钩子 · 合计不超8秒
痛点直给——说清「为什么现在的方式错」
一两句说明问题出在哪，锋利度对准「默认设置/方法/模式」，不对准用户本人。钩子+痛点合计不超过8秒，第8秒必须出现App界面/操作画面。超过8秒还没出画面=失败。
「但选对了≠记住了」「大多数人只用了第一种，还用的是默认设置」
演
中段主体 · 连续路径
操作演示——用户看得到下一步点哪
真实录屏，一步接一步的连续操作路径（首页→背词→小铅笔→自选题型…），不是功能罗列、不是多功能跳转。
「首页点背词 → 右下角点小铅笔 → 进入题型设置 → 换成看单词选假名」
收
结尾 · 具体动作
CTA——给一个能立刻做的动作
收尾给具体行动并回扣开头，不要泛泛「试试看 / 评论区留言」。
✓「收藏这条，今天就去改设置」「来MOJi辞書测一次」
✗「还不快试试」「成功通关！」（泛收束）
📐 叙事结构：AIDA模型（兜底） 真·产品总览、拎不出单一入口痛点时才用 ▼
AIDA 的「兴趣放大 / 欲望铺垫」中段，只在确实是产品总览、或多功能并列、拎不出一个单一入口痛点时才用——它在我们的样本里唯一的战绩是输（慢热开场的「听发音」、多功能总览的「认识MOJi」）。能找出单一痛点的，优先走上面的「痛点直给型」。
A
ATTENTION · 前3秒
钩子——让用户停下来
开头用强钩子，不解释背景，直接进入反常识/痛点/场景。用户通常不会给超过3秒。
✓「别用眼睛背日语单词」「说错了又怎样」
✗「大家好，今天来给大家介绍……」
I
INTEREST · 3–15秒
冲突——让用户想点进去
把问题放大，给用户「我要知道答案」的理由。不要在这里给答案，让用户感受「这说的就是我」。
「背了三遍，第四遍还是空白——你以为是意志力问题，其实是方法问题」
D
DESIRE · 中段
演示——用功能解决冲突
展示功能，但不是列清单，是用功能解决前面提出的冲突。每个功能点对应「这解决了我哪个痛苦」。
「影子跟读让你跟着真人发音逐音节纠正——不靠意志力，靠听觉肌肉记忆」
A
ACTION · 结尾
转化——呼应开头，引导行动
结尾回扣开头钩子，给闭合感。转化语轻，不要强硬推销。
「试试用耳朵背——评论区告诉我你背完了几个词」
🗂️ 备选叙事类型（题型决定） 默认痛点直给，其余按题型选 ▼
类型结构适用切角注意
问题驱动型 提出问题 → 功能回答 → 结尾回扣 反转式 / 痛点式（需要多说几句时） 问题必须真实，不能捏造
场景沉浸型 进入场景 → 遇困难 → 功能出现 → 解决 场景式（多功能/情绪向） 场景要具体，不能是「学日语的时候」
功能清单型 数字引出 → 逐条展示 → 收藏引导 干货式 各条之间要有逻辑联系，不能纯堆砌
🧠 脚本写作原则 4条留住用户的核心规则 ▼
① 中段牵引，结论句可以顿住
中段每句不用句号收尾，用问句或省略号留悬念，让用户有理由看下一段。
但关键结论句/punchline 反而要用句号或感叹号顿住，制造重音——这是高数据视频的共同做法，别一刀切。
✓ 中段：「他们在聊什么？」「还有个地方，很多人没注意到——」
✓ 结论句：「是你一直在用最没用的方式背。」「太爽了吧！」（故意顿住，砸下来）
✗ 中段普通陈述句用句号：「你想找的感觉，这里都有。」（用户可以走了）
② 开头钩子与后续内容不重复同一情绪
钩子说了结论，后面再说同一个结论，用户2秒跳出。
钩子的作用是制造「咦？」，不是给答案。答案留在中段功能展示里。
✓ 钩子：「一个人学日语，累吗？」→ 中段：给出MOJi圈这个答案
✗ 钩子：「一个人学日语撑不下去」→ 下一句：「一个人扛太难了」（重复）
③ 功能展示路径：用户心理 → 触发动作 → 功能出现
不是功能带着用户走，而是用户的情绪和需求触发功能出现。
每个功能点对应一个心理状态，顺序跟着用户的真实使用路径走。
✓ 想放弃了（心理）→ 打开MOJi圈（动作）→ 看到在线人数（功能）
✗ 先介绍发帖，再介绍标签，再介绍船友榜（功能清单顺序）
④ 画面与旁白情绪对应
画面在说一件事，旁白在说另一件，用户会出戏。
画面的视觉情绪和旁白的文字情绪要一致。
✓ 猫咪趴桌上睡着（视觉：累）+ 旁白：「背着背着就睡着了」
✗ 猫咪发呆（视觉：孤独）+ 旁白：「累吗？」（情绪不对）
🎙️ 旁白规范 要做到 vs 避免 ▼
✅ 要做到
第一句从6种钩子里选一种直接进：互动指令 / 质问否定 / 结果先行 / 对比反差 / 身份标签 / 数字利益
钩子+痛点合计不超8秒，第8秒必须出现App画面
说人话，不用书面语或产品用语
短句为主，大量3-8字碎句推进，单句别超15字（高数据样本平均约7字）
功能名嵌入句子，不单独念出
有停顿节奏，便于配合录屏剪辑
首尾闭环，结尾回扣开头钩子
❌ 避免
开头是数据陈述/产品定义/寒暄（「有20%的用户」「其实就是一款…」「大家好」）
产品名当段落标签一个个罗列（反例：Kana / 辞書 / 阅读 逐个报名）
一句话超过30字，气口难断
功能点没有对应画面，旁白在飘
产品说明书语气：「MOJi辞書的词汇记忆功能支持……」
🎬 画面呈现方式 实拍真机 / 录屏样机 / 纯动画 ▼
三条高数据视频正好各用一种，数据排序：实拍真机（102赞）> 录屏样机（69赞）> 纯动画（53赞）。纯动画那条的封面也是唯一的插画图、同样最低——「离真实 App 画面越远，数据越低」在封面和视频里一致。默认偏实拍真机，纯动画作为最后选项。
实拍真机 推荐
拿相机拍真实设备屏幕（手机/电脑/平板），画面有设备、手、真实环境，像真人真的在用。任何功能都能用，包括纯软件设置/题型。→ 自创背词列表，数据最高。
适合：想要自来水/真实感，或多端展示（手机·电脑·平板都能用）。
录屏样机
软件内干净录屏，套进设备样机框，画面干净、信息清楚、制作最省事。→ 默认背词。
适合：操作步骤多、需要看清每一步点哪。
纯动画 谨慎
不用真实界面，用插画/动效/符号表达（放大镜、假名飘动那种）。视觉最自由，但离「真实证据」最远，数据也最低。→ 词汇量测试。
